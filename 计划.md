# 五子棋游戏功能增强执行计划

## 目标 Issue

- **Issue #7**: 添加游戏统计功能
- **Issue #8**: 添加计时器功能

## 一、Issue #7: 游戏统计功能

### 1.1 功能需求
- 记录黑棋/白棋胜场数
- 记录总对局数
- 在界面上展示统计数据
- 提供「重置统计」按钮

### 1.2 实现方案

#### 1.2.1 数据存储
- 使用 `localStorage` 存储统计数据
- 存储键名: `gomoku_stats`
- 数据结构:
  ```javascript
  {
    blackWins: number,  // 黑棋胜场
    whiteWins: number,  // 白棋胜场
    totalGames: number, // 总对局数
    draws: number       // 平局数
  }
  ```

#### 1.2.2 统计管理器 (StatsManager)
- `init()`: 初始化，加载已有数据
- `recordWin(winner)`: 记录胜利
- `recordDraw()`: 记录平局
- `getStats()`: 获取统计数据
- `reset()`: 重置统计
- `save()`: 保存到 localStorage

#### 1.2.3 UI 实现
- 在游戏界面添加统计面板
- 显示：黑棋胜场、白棋胜场、平局、总对局
- 添加「重置统计」按钮
- 在模式选择界面也显示统计入口

### 1.3 修改文件
- `main.js`: 添加 StatsManager，修改游戏结束逻辑
- `index.html`: 添加统计面板 HTML 结构
- `style.css`: 添加统计面板样式

## 二、Issue #8: 计时器功能

### 2.1 功能需求
- 可选的总时间限制（如每方 10 分钟）
- 或每步时间限制（如 30 秒内必须落子）
- 超时判负
- 显示双方剩余时间

### 2.2 实现方案

#### 2.2.1 计时模式
- **总时间模式**: 每方有固定总时间（5/10/15分钟可选）
- **每步时间模式**: 每步必须在指定时间内落子（30/60/90秒可选）
- **无限制**: 默认模式，不计时

#### 2.2.2 计时器管理器 (TimerManager)
- `init(mode, config)`: 初始化计时器
- `start()`: 开始计时
- `stop()`: 停止计时
- `switchPlayer()`: 切换玩家计时
- `getTime(player)`: 获取玩家剩余时间
- `reset()`: 重置计时器
- `isTimeout()`: 检查是否超时

#### 2.2.3 UI 实现
- 在游戏界面添加计时器显示区域
- 显示双方剩余时间（分:秒格式）
- 时间紧迫时红色警告
- 游戏设置界面添加计时模式选择

### 2.3 修改文件
- `main.js`: 添加 TimerManager，修改游戏逻辑
- `index.html`: 添加计时器显示 HTML 结构，计时模式选择
- `style.css`: 添加计时器样式

## 四、注意事项

- 统计数据需要在游戏结束时更新（胜利/平局）
- 计时器需要在玩家切换时正确切换
- 计时器需要支持暂停（如 AI 思考时暂停玩家计时）
- 超时判负时需要清除存档，更新统计
- UI 需要支持暗黑模式
- 需要支持移动端响应式布局
